## Objetivo do projeto
Este projeto tem como objetivo organizar, documentar e preparar a base de dados escolhida para análise, seguindo os critérios definidos para a Etapa 1 da disciplina.

## Estrutura do repositório

### DOC
Pasta com a documentação do projeto.

#### 01_planejamento
Contém os arquivos de planejamento inicial do projeto, como o cronograma da equipe e a organização das etapas.

#### 02_referencias
Contém materiais de apoio, referências e arquivos auxiliares utilizados no desenvolvimento do projeto.

Além disso, na pasta `DOC` também está o documento principal:
- `Detalhamento_do_Projeto.docx`

### CODE
Pasta com os arquivos relacionados à parte prática e aos dados do projeto.

#### dados brutos
Contém os dados originais, sem tratamento, da forma como foram obtidos na fonte.

#### dados tratados
Contém os dados já organizados, limpos ou preparados para uso nas análises.

#### notebooks
Contém os notebooks utilizados para exploração, testes e análises dos dados.

#### scripts
Contém scripts auxiliares e códigos usados no projeto.

## Dataset
O projeto utiliza dados referentes à arrecadação do Sistema de Controle de Receitas Estaduais (SCRE), disponibilizados em arquivos mensais no formato compactado. Esses dados representam informações públicas e permitem acompanhar a evolução da arrecadação ao longo do tempo, possibilitando análises comparativas, identificação de padrões e observação de tendências.

Os arquivos atualmente disponíveis no repositório correspondem a diferentes meses do ano de 2025, organizados na pasta `CODE`. A base foi escolhida por permitir um trabalho de organização, documentação e futura análise exploratória, além de apresentar relevância prática para estudo de dados públicos.
